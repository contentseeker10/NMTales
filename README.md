# 🎮 NMTales

> **Перетвори підготовку до іспитів на захопливий квест!** 🚀  
> **NMTales** — це інтерактивна гейміфікована платформа, створена для учнів 10–11 класів, які готуються до Національного мультипредметного тесту (НМТ). Поєднуйте класичні ігрові механіки (RPG) та розумне навчання з ШІ-персонажами!

---

## ✨ Про проєкт

**NMTales** поєднує якісний навчальний контент із перевіреними ігровими механіками:
* **RPG-елементи**: Очки досвіду (XP), рівні, квести та прокачування характеристик персонажа.
* **Природне спілкування з NPC**: Завдяки інтеграції великих мовних моделей (LLM Gemini), ігрові персонажі реагують на ваші повідомлення та допомагають розбирати складні теми.
* **База завдань НМТ**: Проходження тестів, орієнтованих на офіційну програму тестування.
* **Інтерактивний блокнот**: Збереження важливих нотаток та порад від ШІ-тьютора прямо у грі.

---

## 🏗️ Структура репозиторію

Проєкт складається з двох основних частин:
1. **Клієнт гри (Godot Engine)**: Розташований у папці `game/`.
2. **Серверна частина (Бекенд на .NET 8)**: Розташована у папці `game/backend/`.

---

## 🛠️ Системні вимоги

Перед початком переконайтеся, що на вашому комп'ютері встановлено:
* [**.NET 8.0 SDK**](https://dotnet.microsoft.com/download/dotnet/8.0) (для збірки та запуску бекенду)

## 🚀 Швидкий старт для розробника та тестера

Дотримуйтесь наступного покрокового керівництва для розгортання всього оточення:

### 1️⃣ Налаштування та запуск бази даних (Docker)
Ми використовуємо **PostgreSQL** як основну базу даних. Для швидкого запуску використовується **Docker Compose**:

1. Перейдіть до папки бекенду у вашому терміналі:
   ```bash
   cd game/backend
   ```
2. Запустіть контейнер бази даних:
   ```bash
   docker compose up -d
   ```
   *Це завантажить та запустить офіційний образ PostgreSQL на стандартному порту `5432`.*
3. Перевірити статус роботи контейнера можна командою:
   ```bash
   docker compose ps
   ```

---

### 2️⃣ Конфігурація ШІ-Тьютора (Gemini API Key)

> [!IMPORTANT]
> Для того щоб ШІ-асистенти в грі могли розмовляти та відповідати на запитання користувача, необхідно підключити **Gemini API Key**. Без цього чат із персонажами повертатиме помилки.

1. Отримайте безкоштовний або платний ключ API в [Google AI Studio](https://aistudio.google.com/).
2. Перейдіть до папки проекту бекенду:
   ```bash
   cd game/backend/NMTales.Backend
   ```
3. Налаштуйте ключ у безпечному локальному сховищі конфігурації (.NET User Secrets), щоб не «засвітити» його у коммітах Git:
   ```bash
   dotnet user-secrets set "Gemini:ApiKey" "ВАШ_ОТРИМАНИЙ_КЛЮЧ_API"
   ```
   *За бажанням, ви також можете налаштувати його в файлі `appsettings.json` або `appsettings.Development.json` у секції `"Gemini": { "ApiKey": "ВАШ_КЛЮЧ" }`.*

---

### 3️⃣ Запуск API-бекенду (.NET 8)
1. Перебуваючи у папці `game/backend/NMTales.Backend`, виконайте команду відновлення залежностей та запуску:
   ```bash
   dotnet run
   ```
   * **Авто-міграція та наповнення (Seed)**: При першому запуску сервер самостійно підключиться до Docker-контейнера PostgreSQL, створить структуру бази даних, застосує всі необхідні міграції EF Core та наповнить її початковими тестами, квестами та налаштуваннями.
   * **Адреса сервера**: Локальний API за замовчуванням прийматиме HTTP-запити за адресою `http://localhost:5142`.
   * **Swagger UI (Інтерактивна документація)**: Доступна за адресою [http://localhost:5142/swagger](http://localhost:5142/swagger). Тут можна вручну тестувати роботу API.

> [!TIP]
> Якщо раніше ви використовували хмарну базу даних (наприклад, Neon) та зберігали ConnectionString у User Secrets, видаліть його, щоб сервер підключався саме до локального Docker:
> `dotnet user-secrets remove ConnectionStrings:DefaultConnection`

---

### 4️⃣ Запуск клієнта гри (Godot)
1. Запустіть **Godot Engine v4.6+**.
2. Оберіть **Import** (Імпортувати проєкт) та вкажіть шлях до файлу [project.godot](file:///d:/Development/GameDev/NMTales/game/project.godot) у папці `game/`.
3. Запустіть сцену гри кнопкою **Play** (або натиснувши **F5**).
4. За замовчуванням клієнт налаштований на підключення до `http://localhost:5142` (налаштовується у файлі [network_manager.gd](file:///d:/Development/GameDev/NMTales/game/autoloads/network_manager.gd)).
5. Пройдіть швидку реєстрацію прямо у грі (вигадайте новий нікнейм та пароль) і починайте тестування!

---

## 🧪 Запуск автоматичних тестів бекенду
Для перевірки коректності роботи серверного коду та інтеграції з базою даних, ви можете запустити юніт та інтеграційні тести:
1. Перейдіть до папки тестів або запустіть команду з кореня бекенду:
   ```bash
   dotnet test game/backend/NMTales.Backend.Tests/NMTales.Backend.Tests.csproj
   ```

---

## 🔧 Розробка схем БД та міграції
Якщо ви змінюєте C# моделі даних:
1. Перейдіть до директорії `game/backend/NMTales.Backend`.
2. Створіть міграцію:
   ```bash
   dotnet ef migrations add <НазваМіграції>
   ```
3. Схема БД у контейнері оновиться автоматично при наступному старті сервера.

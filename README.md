# 🎮 NMTales

> **Перетвори підготовку до іспитів на захопливий квест!** 🚀  
> **NMTales** — це інтерактивна гейміфікована платформа, створена для учнів 10–11 класів, які готуються до Національного мультипредметного тесту (НМТ). Поєднуйте класичні ігрові механіки (RPG) та розумне навчання з ШІ-персонажами!

---

## ✨ Про проєкт

**NMTales** поєднує якісний навчальний контент із перевіреними ігровими механіками:
* **RPG-елементи**: Очки досвіду (XP), рівні, квести та прокачування характеристик персонажа.
* **Природне спілкування з NPC**: Завдяки інтеграції великих мовних моделей (LLM Gemini), ігрові персонажі реагують на ваші повідомлення та допомагають розбирати складні теми.
* **База завдань НМТ**: Проходження тестів, орієнтованих на офіційну програму тестування.
* **Інтерактивний блокнот**: Збереження важливих нотаток та порад від ШІ-тьютора прямо у грі.

---

## 🏗️ Структура репозиторію

Проєкт складається з двох основних частин:
1. **Клієнт гри (Godot Engine)**: Розташований у папці `game/` (поточна папка).
2. **Серверна частина (Бекенд на .NET 8)**: Розташована у папці `game/backend/`.

---

## 🛠️ Системні вимоги

Перед початком переконайтеся, що на вашому комп'ютері встановлено:
* **.NET 8.0 SDK** (для запуску бекенду)
* **Docker Desktop** (для локального запуску бази даних PostgreSQL)
* **Godot Engine v4.x** (для запуску клієнту гри)

---

## 🚀 Швидкий старт для розробника

Виконайте ці прості кроки, щоб запустити локальне оточення:

### 1️⃣ Налаштування та запуск бази даних (Docker)
Ми використовуємо **PostgreSQL** як основну базу даних. Для швидкого запуску використовується **Docker Compose**:

1. Перейдіть до папки бекенду:
   ```bash
   cd backend
   ```
2. Запустіть базу даних у фоновому режимі:
   ```bash
   docker compose up -d
   ```
   *Це запустить контейнер PostgreSQL на порту `5432`.*

---

### 2️⃣ Запуск API-бекенду (.NET 8)
1. Перейдіть до папки проекту бекенду:
   ```bash
   cd backend/NMTales.Backend
   ```
2. Запустіть сервер:
   ```bash
   dotnet run
   ```
   * При першому запуску сервер автоматично створить базу даних, застосує міграції Entity Framework Core та наповнить її початковими даними (Seed).
   * Локальний API працюватиме на `http://localhost:5142`.
   * Документація **Swagger UI** буде доступна за адресою `http://localhost:5142/swagger`.

> [!TIP]
> Якщо у вас раніше були налаштовані **User Secrets** для підключення до зовнішніх БД (наприклад, Neon), видаліть їх для роботи з Docker:
> `dotnet user-secrets remove ConnectionStrings:DefaultConnection`

---

### 3️⃣ Запуск клієнта гри (Godot)
1. Відкрийте **Godot Engine**.
2. Імпортуйте проєкт, обравши файл [project.godot](file:///d:/Development/GameDev/NMTales/game/project.godot) з цієї папки.
3. Натисніть кнопку **Play** або клавішу **F5**, щоб запустити гру.
4. Пройдіть швидку реєстрацію (потрібні лише нікнейм та пароль) та починайте гру!

---

## 🔧 Розробка та міграції

Якщо ви вносите зміни в моделі бази даних:
1. Перейдіть у `backend/NMTales.Backend`.
2. Створіть нову міграцію:
   ```bash
   dotnet ef migrations add <НазваМіграції>
   ```
3. База даних автоматично оновиться при наступному запуску додатку або вручну через:
   ```bash
   dotnet ef database update
   ```

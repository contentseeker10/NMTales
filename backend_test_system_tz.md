# Техническое задание: Реализация Системы Тестов (ASP.NET Core)

## 1. Цель
Реализовать серверную логику для прохождения тестов по Математике (Алтари) и Украинскому языку (Свитки) с авторизацией игроков. Система должна быть безопасной (защищенной от читерства), поэтому используется **Stateful-подход** (состояние прохождения теста хранится и валидируется на сервере).

---

## 2. Обновление моделей данных (Entity Framework)

### A. Модель вопроса и ответов (`Question` & `Answer`)
Нужно расширить сущность `Question`, чтобы она поддерживала разделение на темы, хранение путей к изображениям (для LaTeX-формул) и структуру для Drag & Drop тестов по украинскому языку.

```csharp
// enums/Subjects.cs
public enum Subject
{
    Math,
    Ukrainian,
    History
}

// Models/Question.cs
public class Question
{
    public int Id { get; set; }
    public Subject Subject { get; set; }
    public string Topic { get; set; } = string.Empty; // Например, "Logarithms", "Fractions", "Nouns"
    public string Text { get; set; } = string.Empty;  // Текст вопроса (может содержать плейсхолдеры вроде "У лісі росла [0] ялинка...")
    public string? ImagePath { get; set; }            // Путь к картинке в wwwroot (например, "/images/math/eq1.png")
    
    public List<Answer> Answers { get; set; } = new();
}

// Models/Answer.cs
public class Answer
{
    public int Id { get; set; }
    public int QuestionId { get; set; }
    public Question? Question { get; set; }
    public string Text { get; set; } = string.Empty; // Текст ответа / перетаскиваемый элемент
    
    // Для математики:
    public bool IsCorrect { get; set; } 

    // Для украинского языка (Drag & Drop):
    // Указывает индекс плейсхолдера [x] в тексте вопроса, куда нужно перетащить этот ответ.
    // Если null — это дистрактор (неправильный вариант для усложнения).
    public int? CorrectSlotIndex { get; set; } 
}
```

### B. Сессия теста (`UserTestSession`)
Для отслеживания прогресса прохождения теста в реальном времени. При провале или перезапуске сессия обновляется/сбрасывается.

m    public bool IsFailed { get; set; }

## 3. Эндпоинты API (`TestController`)

Все эндпоинты должны быть защищены авторизацией `[Authorize]`.

### 1. Начать тест: `POST /api/test/start`
**Входные данные:**
```json
{
  "subject": "Math",
  "topic": "Logarithms"
}
```
**Логика работы:**
1. Найти активную незавершенную сессию пользователя. Если она есть — можно удалить старую или сбросить её.
2. Выбрать из БД случайные вопросы по указанному `Subject` и `Topic`:
   * Для `Math`: ровно 3 случайных вопроса.
   * Для `Ukrainian`: ровно 1 случайный вопрос (свиток).
   * **Важное требование (без повторов):** 
     * В рамках одной сессии выбранные вопросы должны быть строго **уникальными** (без дубликатов).
     * *Дополнительно (опционально):* Для повышения качества обучения рекомендуется выбирать те вопросы по данной теме, на которые этот пользователь еще не отвечал правильно в предыдущих успешных сессиях.
3. Создать запись `UserTestSession`:
   * Записать `QuestionIds`.
   * `CurrentQuestionIndex = 0`.
   * `RemainingAttempts = 2` (для математики).
4. Вернуть клиенту информацию о текущем (первом) вопросе. **Внимание:** варианты ответов в JSON не должны содержать признаков правильности (`IsCorrect` и `CorrectSlotIndex` должны быть скрыты в DTO, отправляемом клиенту).

**Успешный ответ (200 OK):**
```json
{
  "sessionId": 42,
  "subject": "Math",
  "topic": "Logarithms",
  "currentQuestionIndex": 0,
  "totalQuestions": 3,
  "question": {
    "id": 105,
    "text": "Розв'яжіть рівняння:",
    "imagePath": "/images/math/log_eq1.png",
    "answers": [
      { "id": 401, "text": "1" },
      { "id": 402, "text": "2" },
      { "id": 403, "text": "4" },
      { "id": 404, "text": "8" }
    ]
  }
}
```

### 2. Отправить ответ: `POST /api/test/submit`
Принимает ответ игрока на текущий вопрос сессии и производит валидацию на стороне сервера.

#### Сценарий А: Тест по Математике
**Входные данные:**
```json
{
  "sessionId": 42,
  "answerId": 403
}
```
**Логика сервера:**
1. Проверить существование сессии, принадлежность её пользователю, а также что сессия не провалена и не завершена.
2. Сверить `answerId` с правильным ответом в БД для текущего вопроса.
3. **Если ответ правильный:**
   * Увеличить `CurrentQuestionIndex` на 1.
   * Сбросить `RemainingAttempts = 2` для следующего вопроса.
   * Если это был последний вопрос (`CurrentQuestionIndex == totalQuestions`):
     * Начислить пользователю опыт (XP) в БД (например, +100 XP).
     * Повысить уровень, если накоплено достаточно XP (формула: `(Level + 1) * 100`).
     * Пометить сессию как `IsCompleted = true`.
     * Сохранить изменения и вернуть `{ "correct": true, "completed": true }`.
   * Если есть следующий вопрос:
     * Вернуть `{ "correct": true, "completed": false, "nextQuestion": { ... } }`.
4. **Если ответ неправильный:**
   * Уменьшить `RemainingAttempts` на 1.
   * Если `RemainingAttempts <= 0`:
     * Пометить сессию как `IsFailed = true`.
     * Вернуть `{ "correct": false, "completed": false, "failed": true }`. *(Клиент примет это и заспавнит врагов)*.
   * Если попытки еще есть (осталась 1):
     * Вернуть `{ "correct": false, "completed": false, "failed": false, "remainingAttempts": 1 }`. *(Клиент покажет предупреждающий текст на экране)*.

---

#### Сценарий Б: Тест по Украинскому Языку (Свиток Drag & Drop)
**Входные данные:**
Игрок сопоставляет индексы слотов `[0]`, `[1]` в тексте с перетащенными `answerId`.
```json
{
  "sessionId": 43,
  "slots": [
    { "slotIndex": 0, "answerId": 501 },
    { "slotIndex": 1, "answerId": 505 }
  ]
}
```
**Логика сервера:**
1. Проверить сессию.
2. Для каждого присланного слота проверить по БД: совпадает ли присланный `answerId` с ответом, у которого `CorrectSlotIndex == slotIndex`.
3. Сформировать массив результатов для каждого слота, чтобы клиент мог визуально подсветить элементы (зеленым — правильные, красным — неправильные).
4. Если **все** слоты заполнены верно:
   * Начислить опыт игроку.
   * Пометить сессию как `IsCompleted = true`.
   * Вернуть общий успех и детализацию по слотам.
5. Если есть хотя бы одна ошибка:
   * Сессия сбрасывается (или помечается как `IsFailed`). Свиток сбрасывается на клиенте.
   * Вернуть общий провал и детализацию по слотам (какие конкретно слоты были неверными).

**Пример ответа при частичной ошибке (200 OK):**
```json
{
  "correct": false,
  "completed": false,
  "failed": true,
  "slotResults": [
    { "slotIndex": 0, "isCorrect": true },
    { "slotIndex": 1, "isCorrect": false }
  ]
}
```

## 4. Требования к инфраструктуре и Сидированию (Data Seeding)

1. **Раздача статических файлов:** В `Program.cs` должна быть подключена раздача статики:
   ```csharp
   app.UseStaticFiles(); // Позволит скачивать картинки из папки wwwroot
   ```
2. **Директория картинок:** Создать папку `wwwroot/images/math/` на сервере для хранения формул в формате PNG/SVG.
3. **Seeding в InMemory DB:** Написать инициализатор данных, который при запуске приложения наполняет базу данных:
   * Не менее 5-10 уникальных вопросов по математике на конкретную тему (например, тема "Logarithms" / "Логарифмы") с заполненными `ImagePath`. Это позволит протестировать логику случайного выбора без повторений.
   * 2-3 вопроса по украинскому языку (тема "Syntax" / "Синтаксис") с плейсхолдерами в `Text` и привязанными `Answer` (часть с `CorrectSlotIndex`, часть с `null` в качестве дистракторов).

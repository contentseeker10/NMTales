# ⚡ Бекенд NMTales

Ласкаво просимо до бекенд-сервісу **NMTales**. Це високопродуктивний RESTful Web API, побудований на базі **.NET 8**, який забезпечує автентифікацію користувачів, відстеження прогресу та збереження навчальних питань для ігрової платформи NMTales.

---

## 🚀 Особливості

- **⚡ Легкий та швидкий**: Створений за допомогою ASP.NET Core Minimal APIs & Controllers.
- **🔐 Безпечна авторизація**: JWT Bearer автентифікація та сучасне хешування паролів за допомогою **BCrypt** (вхід лише за юзернеймом, максимально просто, жодної пошти!).
- **💾 База даних в оперативній пам'яті (In-Memory)**: Швидке локальне налаштування БД без додаткових конфігурацій для миттєвої розробки та тестування.
- **📚 Інтерактивна документація API**: Інтеграція зі Swagger UI «з коробки» для зручного тестування ендпоінтів.

---

## 🛠️ Системні вимоги

Для локального запуску бекенду переконайтеся, що у вас встановлено:
* [**.NET 8.0 SDK**](https://dotnet.microsoft.com/download/dotnet/8.0) або новішої версії.

---

## 🏃 Швидкий старт

### 1. Відновлення та збірка
Перейдіть до директорії проєкту бекенду, відновіть залежності та зберіть проєкт:
```bash
cd game/backend/NMTales.Backend
dotnet restore
dotnet build
```

### 2. Запуск сервера
Запустіть сервер у режимі розробки (development):
```bash
dotnet run
```
За замовчуванням сервер запуститься на:
* **HTTPS**: `https://localhost:7142` (або іншому налаштованому порту)
* **HTTP**: `http://localhost:5142`

---

## 🔍 Тестування API

### Swagger UI (Інтерактивне тестування у браузері)
Коли сервер запущено в режимі розробки (`Development`), відкрийте браузер та перейдіть за адресою:
```
http://localhost:5142/swagger
```
Тут ви можете детально вивчити всі доступні ендпоінти (`/api/auth`, `/api/question`) та протестувати їх прямо з браузера.

### Файл HTTP-запитів
Ви також можете скористатися вбудованим файлом [NMTales.Backend.http](file:///d:/Development/GameDev/NMTales/game/backend/NMTales.Backend/NMTales.Backend.http) у вашому IDE (наприклад, VS Code з плагіном REST Client або Visual Studio) для швидкого надсилання готових запитів на реєстрацію та вхід.

## 🏗️ Архітектура проєкту

* **`Controllers/`**: Містить ендпоінти API (`AuthController`, `QuestionController`).
* **`DTO/`**: Об'єкти передачі даних (Data Transfer Objects), що описують структуру запитів та відповідей.
* **`Models/`**: Доменні моделі (`User`, `Question`, `UserProgress`).
* **`Services/`**: Шари бізнес-логіки (`JwtService`).
* **`Data/`**: Контекст бази даних (`ApplicationDbContext`).

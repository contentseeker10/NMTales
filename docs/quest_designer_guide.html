NMTales Quest Guide

Гайд для гейм-дизайнерів з проектування квестів, діалогів та налаштування NPC у грі.

📘 Документація Пайплайну V2.0

1. Створення NPC у Godot

Кожен NPC має бути унікальним об'єктом сцени. Для налаштування переконайтеся, що ви експортували параметр npc_id (наприклад, npc_quest) в інспекторі Godot. Цей ідентифікатор зв'язує фізичного персонажа на карті з його файлами квестів та діалогів.

2. Діалоги та логіка гілкування

Діалоги зчитуються клієнтом динамічно на основі поточного стану квесту. Менеджер діалогів перевіряє стан у такому порядку:

• Якщо квест активний і ціль виконана → завантажується quest_complete.json.
• Якщо квест активний, але не виконаний → quest_progress.json.
• Якщо є наступний не пройдений квест і для нього існує файл пропозиції → завантажується {quest_id}_offer.json.
• В іншому випадку завантажується casual-діалог casual.json.

3. Файл пропозиції квесту (*_offer.json)

Файли пропозицій містять репліки NPC та вибір гравця для старту квесту. Для кнопки згоди необхідно обов'язково налаштувати екшн:

"action": {
  "type": "accept_quest",
  "quest_id": "quest_1"
}

4. Файл завершення квесту (quest_complete.json)

Файл завершення квесту підвантажується автоматично при виконаній цілі. В кнопці підтвердження обов'язково пропишіть екшн:

"action": {
  "type": "complete_quest"
}

5. Серверна конфігурація квесту

Сам квест реєструється на бекенді. Бекенд валідує події, нараховує досвід і підвищує рівень. Файли мають лежати в закритій папці бд на сервері, щоб уникнути датамайнінгу.

NMTales Dialogue Guide

Гайд для гейм-дизайнерів з написання розгалужених діалогів та інтеграції їх із системою квестів.

📘 Документація Діалогів V1.1

1. Загальний принцип дерева діалогу

Діалоги описуються у форматі JSON у вигляді дерева вузлів (Nodes). Кожен вузол має унікальний текстовий ключ (ідентифікатор вузла). За замовчуванням діалог завжди починається з вузла з ключем "start".

Вузол містить репліку NPC ("text") та список варіантів відповіді гравця ("choices").

2. Формат вибору (Choice)

Кожен вибір гравця містить:

• "text" — репліка гравця на кнопці вибору.
• "next_node" — ідентифікатор наступного вузла діалогу. Якщо вибрати значення "exit", то діалог завершиться і вікно закриється.
• "action" (опціонально) — тригер спеціальної дії (наприклад, для квестів).

3. Специфіка квестових діалогів

У системі NMTales діалоги тісно пов'язані зі станом квесту у гравця. Залежно від етапу, гра шукає JSON-файли за певними шляхами:

📌 Типи файлів та шляхи їх збереження:

1. Casual (Звичайний діалог):
Показується, коли у NPC немає доступних чи активних квестів.
💾 Шлях: res://assets/shared/dialogues/{npc_id}/casual.json

2. Offer (Пропозиція квесту):
Завантажується, якщо у NPC є доступний квест, який гравець ще не взяв.
💾 Шлях: res://assets/shared/dialogues/{npc_id}/quests/{quest_id}_offer.json
💡 Обов'язкова дія для взяття: Вузол згоди має містити action: 
"action": {
  "type": "accept_quest",
  "quest_id": "quest_id_here"
}
⚠️ Глобальні квестовi стани (Спільні файли):

3. Progress (Квест виконується):
Завантажується автоматично, коли у гравця є активний квест від цього NPC, але умови ще не виконані.
💾 Шлях: res://assets/shared/dialogues/quest_progress.json

4. Complete (Квест виконано!):
Завантажується, коли гравець виконав умови активного квесту і прийшов здавати його NPC.
💾 Шлях: res://assets/shared/dialogues/quest_complete.json
💡 Обов'язкова дія для завершення: Кнопка завершення має містити action: 
"action": {
  "type": "complete_quest"
}

4. Специфікація вузла дії (Action Node)

Клієнт гри реагує на події в діалогах. Доступні два ключові екшени:

• "accept_quest" — бере квест у бд на сервері та активує його на клієнті.
• "complete_quest" — завершує квест на сервері, видає нагороди (XP/Level) та оновлює HUD гравця.
